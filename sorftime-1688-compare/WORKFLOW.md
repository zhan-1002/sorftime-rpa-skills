# 选品工作流设计

## 策略定位

**精铺策略**：不追求蓝海垄断或爆款日出百单。寻找有明确生态位的小产品——月销 50-100 单、评论壁垒低、自然需求驱动、1688 采购可控。

三个生存路径：
- **差异化**：我能做得比你好（水果造型杯 vs 普通玻璃杯）
- **细分需求**：你忽略了这个需求（加高折叠椅 vs 标准高度）
- **季节流量**：我只卖两个月，高毛利覆盖短窗口（圣诞袜架 198% ROI）

## 六维判断模型

基于 10 个验证样本的数据画像：

| 维度 | 强信号 | 中信号 | 弱信号 | 权重 |
|---|---|---|---|---|
| 竞品评价中位数 | < 130 条 | 130-500 | > 500 | 25% |
| 广告依赖度（广告组数） | <= 5 组 | 6-12 组 | > 12 组 | **25%** |
| 新品存活证据 | Top20 有上架<1年 | - | 无新品 | 15% |
| 1688 可达性 | 图搜命中同款 | 搜到组件 | 无结果 | 15% |
| ROI | > 120% | 80-120% | < 80% | 10% |
| 差异化空间 | 竞品评分<4.2 | 4.2-4.5 | > 4.5 | 10% |

### 广告依赖度（关键维度）

```
广告组数 / 月销量的比值 = 每单所需的广告投入

螃蟹头箍：3组 / 100单 = 0.03  ← 自然需求驱动，精铺最优
折叠椅： 37组 / 100单 = 0.37  ← 广告驱动，红海内卷

来源：sellersprite 广告洞察 (/v3/ads-insights?q=<ASIN>&months=6)
```

## 完整流程

```
Step 1. 品类来源
  ├─ 样本 ASIN 的品类（已通过验证）
  └─ sellersprite markets 关键词搜索 → category_filter 过滤

Step 2. 产品发现
  └─ Amazon 多关键词搜索 → 提取 ASIN → 标题验证

Step 3. 竞争分析（sellersprite competitors）
  ├─ 价格分布（P10/P25/P50/P75/P90）
  ├─ 评价分布（中位数 + 分桶）
  ├─ 品牌集中度（Top5 占比）
  ├─ 新入者统计（<30 评产品数）
  └─ FBA / 中国卖家占比

Step 4. 模板选定
  └─ 低评 + 低价位 + 可差异化 → 模板 ASIN

Step 5. 广告分析（sellersprite ad-insights）
  ├─ 广告组数量 → 广告依赖度
  └─ SP / SBV 类型分布

Step 6. 深度分析（Amazon 页面 + sellersprite）
  ├─ 套装内容拆解（件数/材质/尺寸/重量）
  ├─ 评论情感（好评关键词 / 差评痛点）
  └─ 产品图片（用于 1688 图搜）

Step 7. 1688 图搜比价
  ├─ 主图 → 1688 图搜 → 供应商列表
  ├─ 组件拆解成本估算
  └─ FBA 利润计算

Step 8. 差异化方案 + 利润重算
  ├─ 基于差评痛点 + 成本优势
  └─ 重新计算差异化后利润
```

## 已验证的样本规律

| 信号 | 通过品特征 | 来源 |
|---|---|---|
| 价格带 | $12-$44，中位 $29 | 10 ASIN |
| 评价数 | 2-130 条，中位 28 | sellersprite competitors |
| 竞品评价中位 | < 130 条 | Step 3 竞争分析 |
| 广告依赖 | 3-7 组 = 优，>12 组 = 慎 | Step 5 广告洞察 |
| FBA | 84-95%，非区分信号 | competitors |
| 中国卖家 | 84%，非区分信号 | competitors |
| 自然需求品 | 广告组/月销 < 0.05 | 螃蟹头箍、汤碗、圣诞袜架 |
